What this tool does

                
The WCGD applet is the authoring surface for the prompt library and wait-state phrases that power GenAIsys. You define what users can ask and how the system frames the criteria while assisting.

                
Output is a single Gen20 JSON file consumed by the GenAIsys UI client. Edit here, save to your browser, then download the JSON when you're ready to ship.


                
How it's organized

                
    
                    
  • Categories group related prompts (left sidebar).
    • 
                    
  • Sub-categories divide a category into themes.
    • 
                    
  • Prompts are the actual questions users can ask.
    • 
                    
  • Wait-state phrases are the messages cycled while GenAIsys responds.
    • 
                


                
First time here?

                
Nothing leaves your browser until you download the JSON. Edit anything, save snapshots locally, and export when you're ready to ship.

Vocabulary

                

                    
Category
Top-level grouping of related prompts. Shown in the sidebar tree.

                    
Sub-category
A theme inside a category. Selecting one reveals its prompts in the main panel.

                    
Prompt
The plain-language question users see. No tokens.

                    
Caption
Short display label (≤ ~30 chars) used in suggestion chips and lists.

                    
Template
The same question with ##TOKENS## swapped in for dynamic values. Filled by the system at runtime.

                    
Token
One of nine placeholders: ##HCP_NAME##, ##HCP_HCO##, ##PRODUCT##, ##TIME_PERIOD##, ##ZIP_CODE##, ##INDICATION##, ##SPECIALTY##, ##COMPETITOR##, ##FRANCHISE##. Note: ##HCP_NAME## is for Healthcare Professionals only, while ##HCP_HCO## accepts both Healthcare Professionals (HCP) and Healthcare Organizations (HCO).

                    
Criterion
A data input the prompt expects. Each is one of the nine token types and has three flags.

                    
Required
The prompt cannot run until this value is supplied (or asked for).

                    
Match Tap
Bind to whatever the user tapped/selected (e.g., the HCP they were viewing) instead of asking.

                    
Default
Fallback value used when the criterion is neither supplied nor tapped.

                    
Franchise
Therapeutic area the content belongs to (e.g., IMMUNOLOGY - DERM). Scopes visibility — empty list = visible to all.

                    
Keywords
Search hints that help GenAIsys match natural-language queries to this prompt.

                    
Workflow ID
Optional backend handler name (e.g., snapshot). Blank = default routing.

                    
Feature Flag
Optional gate name for staged rollouts. Blank = always on.

                    
RM Favorite
Regional Manager Favorite — prompts designated by management to appear in users' suggested strip. Unlike user favorites (which users set themselves), RM Favorites are set centrally in this config. Use sparingly for the highest-value, broadly-useful prompts.

                    
Home Page
Marks a prompt to appear in the home page carousel. Prompts are only shown if they are configured as available to the current user's franchise.

                    
Wait-state phrase
Reassuring/loading message cycled while the AI responds. Empty franchise list = global.

                


                
Franchise scoping

                
Franchises can be set in three places, each narrowing the audience:

                
    
                    
  1. Category franchises — gate the whole category.
    2. 
                    
  2. Prompt franchises — override category scope for that prompt.
    3. 
                    
  3. Wait-state franchises — make a phrase appear only for those franchises.
    4. 
                

                
Empty in any of those three places = "applies to everyone" at that level.

            


            

                
Authoring a prompt — start to finish

                
    
                    
  1. Pick or create a category. Use the sidebar's + Add Category, or click an existing category's pencil icon to edit it.
    2. 
                    
  2. Add a sub-category when creating/editing the category, or skip it and add more later. Set the Order field (defaults to 0) to control display order — lower numbers appear first.
    3. 
                    
  3. Select the sub-category in the sidebar — the main panel switches to its prompt list.
    4. 
                    
  4. Click + Add Prompt in the prompt list header.
    5. 
                    
  5. Caption + Prompt first — short label, then the natural-language question.
    6. 
                    
  6. Template — restate the question with ##TOKENS##. Click chips to insert.
    7. 
                    
  7. Order — set the numeric order for this prompt (defaults to 0). Lower numbers appear first in lists.
    8. 
                    
  8. Criteria — for each token in the template, add a criterion row. Set Required, Match Tap, and a Default when relevant.
    9. 
                    
  9. Franchises — leave empty to inherit the category, or pick specific ones to narrow scope.
    10. 
                    
  10. Keywords — add 3–6 search terms users might type when looking for this prompt.
    11. 
                    
  11. Save Prompt — the JSON output below refreshes immediately.
    12. 
                


                
Authoring a wait-state phrase

                
    
                    
  1. Expand the Wait State Phrases panel near the bottom.
    2. 
                    
  2. Click + Add Phrase. A new row appears with an empty input.
    3. 
                    
  3. Type the phrase. Keep it short and reassuring (e.g., "Pulling the latest TRx data...").
    4. 
                    
  4. Pick franchises if it's franchise-specific. Leave empty for a global phrase.
    5. 
                


                
Editing order

                
    
                    
  • Sub-category order: Click the pencil icon (✏️) next to any sub-category in the sidebar to edit both its name and order.
    • 
                    
  • Prompt order: Click Edit on any prompt card to modify its order field along with other properties.
    • 
                    
  • Order values are numeric — lower numbers appear first. Use gaps (0, 10, 20...) to make future reordering easier.
    • 
                


                
Tips

                
    
                    
  • Match every ##TOKEN## in your template with a criterion of the same name.
    • 
                    
  • Captions should read like a chip label, not a sentence.
    • 
                    
  • Use Match Tap when the user is already looking at the entity in question — it spares them from re-typing.
    • 
                    
  • Save to Browser often — versions auto-prune at 20.
    • 
                

            


            

                
How your work is stored

                
Everything stays in your browser. There is no server. The header buttons are your save/restore controls:

                

                    
Load Sample Data
Replaces current state with a realistic example config. Useful for exploration.

                    
Load from Clipboard
Paste-and-import. Reads the system clipboard, expects a Gen20 JSON object.

                    
Import JSON
File-picker import. Accepts a .json file in Gen20 format.

                    
Save to Browser
Snapshot the current state into localStorage. Up to 20 most-recent versions are kept; older ones are auto-pruned.

                    
Load from Browser
Pick a saved snapshot to restore. You can also delete old versions from this dialog.

                    
Download JSON
Save the current state to a .json file on disk. This is the artifact you ship.

                


                
Limits and recovery

                
Browser storage is ~5–10 MB depending on the browser. If a save fails with "quota exceeded," delete older versions from Load from Browser and retry. Clearing site data (browser settings) erases all saved versions — download a JSON backup before doing that.

            


            

                
Gen20 schema

                
The full JSON schema is embedded in the page and visible under View JSON Schema in the JSON Output panel. Key constraints:

                
    
                    
  • wcgd and wait_state are both required at the root.
    • 
                    
  • Categories require cat, franchises, roles, sub-cats. Empty arrays are valid; missing keys are not.
    • 
                    
  • Prompts require the full shape: prompt, caption, template, franchises, roles, criteria, wrkf, feature, fav. keywords is the only optional field.
    • 
                    
  • The criterion name is one of eight enums — same list as the token chips.
    • 
                    
  • Franchise is one of eight enums; chip UI dedupes.
    • 
                

                
A "validate against schema" button is on the roadmap — for now, the editor enforces required keys at save time.
